Для разработки хорошего программного продукта надо:
    разделить продукт на составные части и делать их по отдельности (конвейер)


Существуют только сборники опыта разработки (лучшие практики) - нет рецепта идеальной программы или инструкции для её создания

    SWEBOK - про разработку
    PMBOK - про управление проектами


Проблемы при создании ПО
    невозможность договориться внутри команды
    срыв сроков
    окончание поддержки другого ПО
    ушёл ключевой человек
    большое количество ошибок

pull-стратегия - постепенные изменения в процессе создания ПО
push-стратегия - резкое изменение процесса создания ПО (сломать всё и построить заново)


Разработка делится на фазы и этапы:
    этап написания требований
    фаза тестирования

Модели разработки ПО:
    Каскадная (водопадная)
        первая модель
        родилась в военной отрасли
        см. drawio
        требования не меняются после постановки задачи
        пример: контрактная разработка ПО

    V-образная
        см. drawio
        решила проблему поиска ошибок
        не решила проблему потери актуальности продукта к концу разработки

    Инкрементальная модель
        делим программу на куски и разрабатываем их отдельно
        есть возможность возврата на предыдущий этап

    Спиральная модель
        наиболее актуальная на данный момент
